2) V2 Delta Summary (What Improved vs V1)
======================================================================

Verified improvements
- Auth hardening improved:
  - `JWT_SECRET` fallback removed (`src/lib/auth.ts` now fails fast if missing).
  - Admin login moved to DB-backed username/password with bcrypt (`src/actions/admin.ts`).
  - Most admin mutation actions now call `verifyAdminSession()`.
- Design token issue from V1 was fixed:
  - `--card` and `--color-card` tokens now exist (`src/app/globals.css`).
  - `bg-card` utilities are now generated in compiled CSS.
- User/admin UX progress:
  - Category filter/sort is now wired (`CategoryToolFilter`, `CategorySort`).
  - Prompt page improved with metadata, structured data, related prompts, comparison slider.
  - Prompt pagination added in admin prompts page.
  - Prompt status and thumbnail focal point support added.
- Upload endpoint improved:
  - Added max size and MIME allowlist checks.

New/remaining high-risk gaps
- Unsafe raw SQL and missing auth guard in settings actions.
- Global middleware + root layout headers strategy turned almost all routes dynamic.
- Lint/type quality regression is larger than V1.

4) Prioritized Bug and Risk Backlog (V2)
======================================================================

P0-001 - SQL injection risk in settings update path
- Severity: `P0`
- Effort: `M`
- Evidence:
  - `src/actions/settings.ts` uses `$queryRawUnsafe` and interpolates `siteName` directly into SQL.
  - Example pattern: `UPDATE "SystemSettings" ... "siteName" = '${data.siteName}' ...`.
- Impact:
  - Crafted input can break query or inject SQL.
  - Potential data compromise / unauthorized DB mutation.
- Repro:
  1. Enter a site name containing quote payload.
  2. Trigger Save in settings.
  3. Query behavior becomes unsafe/undefined.
- Fix recommendation:
  - Replace raw SQL with Prisma model operations (`prisma.systemSettings.update`).
  - If raw SQL must remain, use parameterized query API only.

P0-002 - Settings mutation action lacks explicit admin auth check
- Severity: `P0`
- Effort: `S`
- Evidence:
  - `updateSystemSettings()` in `src/actions/settings.ts` has no `verifyAdminSession()` guard.
  - Other admin actions already enforce session checks, settings does not.
- Impact:
  - Security boundary inconsistency for a critical operation (maintenance mode + site config).
- Repro:
  1. Inspect action implementation.
  2. Confirm no role/session validation before write.
- Fix recommendation:
  - Add shared `requireAdminSession()` guard in settings actions.

P1-101 - Global middleware + root layout header strategy forces dynamic rendering
- Severity: `P1`
- Effort: `M`
- Evidence:
  - `src/middleware.ts` matcher applies to nearly all paths.
  - `src/app/layout.tsx` uses `headers()` and `x-pathname` to branch behavior.
  - Build output shows most routes now `ƒ` (dynamic).
- Impact:
  - Higher TTFB, reduced cacheability, increased infra cost.
- Repro:
  1. Run `npm run build`.
  2. Observe route table now mostly dynamic.
- Fix recommendation:
  - Restrict middleware scope to admin/auth needs.
  - Move maintenance branching away from global dynamic layout path.

P1-102 - Deprecated middleware convention still active
- Severity: `P1`
- Effort: `S`
- Evidence:
  - Build warning: `"middleware" file convention is deprecated. Use "proxy".`
  - File currently `src/middleware.ts`.
- Impact:
  - Forward compatibility risk with framework updates.
- Fix recommendation:
  - Migrate to `src/proxy.ts` and preserve matcher semantics.

P1-103 - Prompt lookup uses unsafe typed cast and non-unique filter pattern
- Severity: `P1`
- Effort: `M`
- Evidence:
  - `getPromptBySlug()` in `src/actions/prompt.ts` uses:
    - `findUnique({ where: { slug, status: "published" } as any })`
- Impact:
  - Bypasses type safety and can hide runtime query issues.
  - Reliability risk for prompt detail page and metadata generation.
- Fix recommendation:
  - Use `findFirst({ where: { slug, status: "published" } })` or unique-only `findUnique` then status check.

P1-104 - Navbar search deep-link is broken (`q` param ignored)
- Severity: `P1`
- Effort: `S`
- Evidence:
  - Navbar pushes `/search?q=...` (`src/components/layout/Navbar.tsx`).
  - Search page initializes from `tool` param only (`src/app/search/page.tsx`).
- Impact:
  - User enters query in navbar but search page does not prefill/run that query.
- Repro:
  1. Search from navbar.
  2. Land on search page without expected query state.
- Fix recommendation:
  - Read and apply `q` from `useSearchParams()` as primary initial query.

P1-105 - Lint/type quality gate regressed to 99 issues
- Severity: `P1`
- Effort: `M`
- Evidence:
  - `npm run lint` -> 99 issues (74 errors, 25 warnings).
  - Widespread `any`, hook-pattern violations, JSX entity issues, raw img warnings.
- Impact:
  - Lower reliability, harder refactors, hidden runtime defects.
- Fix recommendation:
  - Introduce typed DTOs and strict Prisma return types.
  - Remove `any` casts from actions/pages.
  - Make lint pass mandatory in CI.

P1-106 - Multiple utility classes are referenced but not generated
- Severity: `P1`
- Effort: `S`
- Evidence:
  - Classes used in code: `animate-in`, `fade-in`, `zoom-in`, `slide-in-from-bottom-*`, `prose`, `text-destructive`, `bg-destructive/*`, `custom-scrollbar`.
  - Compiled CSS has no matching utility definitions.
- Impact:
  - Missing animations, missing typography styling, missing destructive color styles.
  - UI appears inconsistent and partially unstyled.
- Fix recommendation:
  - Either add required plugins/token mappings or replace with supported utilities.

P2-201 - Settings page has pseudo-persistent controls
- Severity: `P2`
- Effort: `M`
- Evidence:
  - `baseUrl` and `adminEmail` are editable in UI but not persisted by `updateSystemSettings()`.
  - "Clear System Cache" is local UI toggle only.
- Impact:
  - Misleading admin UX and false operational confidence.
- Fix recommendation:
  - Persist actual fields via typed settings model.
  - Wire real cache-clear action or relabel clearly as demo.

P2-202 - Users management is still mock-only
- Severity: `P2`
- Effort: `L`
- Evidence:
  - `src/app/admin/(dashboard)/users/page.tsx` uses local mock array and fake loading.
- Impact:
  - No real user/role management despite UI presence.
- Fix recommendation:
  - Connect to `Admin` model + role policy + invite/reset flows.

P2-203 - Report modal is mock submission only
- Severity: `P2`
- Effort: `M`
- Evidence:
  - `src/components/ui/ReportModal.tsx` uses timeout-based success and does not send to backend.
- Impact:
  - Users think reports are submitted when no data is stored.
- Fix recommendation:
  - Create report API/model and admin triage queue.

P2-204 - Search page performs unnecessary fetches on empty query
- Severity: `P2`
- Effort: `S`
- Evidence:
  - Empty query path fetches latest prompts, but UI state still shows "Start typing to search."
- Impact:
  - Wasted server work with no user-visible value.
- Fix recommendation:
  - Skip fetch when query is empty, or show curated prompt grid in empty state.

P2-205 - Category detail performs duplicate data fetch
- Severity: `P2`
- Effort: `S`
- Evidence:
  - `src/app/categories/[slug]/page.tsx` calls `getCategoryBySlug` twice (filtered + unfiltered) per request.
- Impact:
  - Extra DB load and slower response on category pages.
- Fix recommendation:
  - Fetch once and derive tool list through optimized query or cached category prompt metadata.

P2-206 - Upload hardening still incomplete
- Severity: `P2`
- Effort: `M`
- Evidence:
  - No magic-byte signature verification.
  - No guaranteed `mkdir` for upload directory.
  - Cookie token parsing still regex-based from header string.
- Impact:
  - Residual spoofing and operational reliability risks.
- Fix recommendation:
  - Add signature validation, `mkdir(..., { recursive: true })`, and safer cookie parsing helpers.

P2-207 - Category management remains unpaginated
- Severity: `P2`
- Effort: `S`
- Evidence:
  - Prompts table now paginated, categories table still loads all rows.
- Impact:
  - Scalability inconsistency for admin management.
- Fix recommendation:
  - Add pagination/search for categories list.

P2-208 - Stylelint is installed but not configured
- Severity: `P2`
- Effort: `S`
- Evidence:
  - `npx stylelint "src/**/*.css"` fails due missing config.
- Impact:
  - CSS quality gate unavailable.
- Fix recommendation:
  - Add `.stylelintrc` + script + CI integration.
