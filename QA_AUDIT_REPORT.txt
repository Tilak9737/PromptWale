3) Prioritized Bug Findings Backlog
======================================================================

P0-001 - Hardcoded auth fallbacks enable insecure production defaults
- Severity: `P0`
- Effort: `S`
- Evidence:
  - `src/lib/auth.ts` uses fallback JWT secret string in `getJwtSecretKey()`.
  - `src/actions/admin.ts` uses fallback `ADMIN_PASSWORD` default (`"promptwale2024"`).
- Impact:
  - If env vars are missing/misconfigured, app can run with predictable credentials.
  - Direct account compromise risk.
- Repro:
  1. Unset `JWT_SECRET` and `ADMIN_PASSWORD`.
  2. Start app and authenticate with fallback password.
- Fix recommendation:
  - Remove all auth fallbacks.
  - Fail fast at startup when required auth env vars are absent.
  - Require strong secret entropy and minimum password policy.

P0-002 - Admin server actions do not enforce auth at action boundary
- Severity: `P0`
- Effort: `M`
- Evidence:
  - `src/actions/admin.ts` (`createPrompt`, `updatePrompt`, `deletePrompt`, `createCategory`, `updateCategory`, `deleteCategory`) performs mutations with no `requireAdmin` check.
- Impact:
  - Route middleware is the only control; server actions themselves are not self-defending.
  - Increases blast radius from action token leakage or future route reuse.
- Repro:
  1. Inspect action implementations and note direct Prisma mutation calls.
  2. Confirm no centralized server-side role assertion before mutation.
- Fix recommendation:
  - Add `requireAdmin()` utility in every admin action.
  - Validate token role in action context before touching DB.
  - Return structured authorization errors for UI handling.

P0-003 - Weak auth model (single shared password, no rate limit, no audit)
- Severity: `P0`
- Effort: `L`
- Evidence:
  - `src/app/admin/login/page.tsx` captures password only.
  - `prisma/schema.prisma` contains `Admin` model but login flow does not use hashed per-user credentials.
- Impact:
  - No user-level accountability.
  - Susceptible to brute force and credential sharing.
- Repro:
  1. Repeated login attempts are allowed without lockout.
  2. No user identity or per-user trace exists in auth flow.
- Fix recommendation:
  - Migrate to account-based admin auth with hashed passwords (`bcrypt/argon2`).
  - Add attempt throttling, lockout, and audit events.
  - Implement session rotation and explicit logout invalidation.

P1-101 - Upload endpoint missing strict file validation and safety limits
- Severity: `P1`
- Effort: `M`
- Evidence:
  - `src/app/api/upload/route.ts` trusts client-supplied filename extension.
  - No MIME magic-byte verification, max size check, dimension constraints, or content scanning hook.
- Impact:
  - Risk of oversized uploads, invalid file types, and storage abuse.
  - Operational and security risk.
- Repro:
  1. Submit large non-image file renamed with image extension.
  2. Endpoint accepts and writes file to `public/uploads`.
- Fix recommendation:
  - Enforce allowlist (`image/jpeg`, `image/png`, `image/webp`).
  - Validate file signature, max bytes, and optional dimension bounds.
  - Reject files with clear structured error messages.
  - Add rate limiting and upload quotas.

P1-102 - Multiple visible controls are non-functional (dead UX)
- Severity: `P1`
- Effort: `M`
- Evidence:
  - Navbar search input has no navigation/search handler: `src/components/layout/Navbar.tsx`.

  - Search "Advanced Filters" is a clickable label only: `src/app/search/page.tsx`.
  - Admin "View All Prompts" button has no action: `src/app/admin/(dashboard)/page.tsx`.
  - Admin "Save as Draft" has no behavior: `src/app/admin/(dashboard)/prompts/new/page.tsx`.
  - Users/Settings are placeholders only.
- Impact:
  - Broken expectations, lower trust, lower conversion.
- Repro:
  1. Click each control above.

- Fix recommendation:
  - Either wire full behavior or hide controls until ready.
  - Add explicit "Coming in vNext" state only where roadmap placeholders are intentional.

P1-104 - Lint baseline is failing (type safety + hook correctness + JSX escapes)
- Severity: `P1`
- Effort: `M`
- Evidence:
  - `npm run lint` -> 51 issues (33 errors, 18 warnings).
  - Key classes:
    - `no-explicit-any` across actions/pages/components.
    - `react-hooks/set-state-in-effect` in navbar theme mount pattern.
    - `react/no-unescaped-entities` in text nodes.
- Impact:
  - Reduced maintainability and release confidence.
- Repro:
  1. Run `npm run lint`.
  2. Observe failing output.
- Fix recommendation:
  - Add strict typed DTOs for actions and view models.
  - Replace problematic hook pattern with hydration-safe theme button strategy.
  - Clean JSX entity escapes and enable CI lint gate.

P1-105 - Missing semantic `card` token causes silent styling gaps
- Severity: `P1`
- Effort: `S`
- Evidence:
  - `src/app/globals.css` defines tokens for `background`, `foreground`, `primary`, `accent`, `muted`, `border`, etc., but no `card` token mapping.
  - Codebase uses `bg-card` widely (`src/app/admin/*`, `src/components/ui/PromptCard.tsx`, others).
  - Compiled CSS (`.next/static/chunks/*.css`) has no `.bg-card` utility rule.
- Impact:
  - Surfaces intended as cards can render without intended background color.
  - Visual hierarchy inconsistency across user/admin screens.
- Repro:
  1. Inspect token definitions and utility generation.
  2. Confirm `bg-card` utility missing in generated CSS.
- Fix recommendation:
  - Add semantic token family:
    - `--card`, `--card-foreground`
    - `--surface-1`, `--surface-2`
  - Map into Tailwind theme tokens (`--color-card`, etc.).

P2-201 - Tool is stored as comma-separated string (query/model fragility)
- Severity: `P2`
- Effort: `L`
- Evidence:
  - `createPrompt` stores `tool: tools.join(", ")` in `src/app/admin/(dashboard)/prompts/new/page.tsx`.
  - Search relies on string `contains` in `src/actions/prompt.ts`.
- Impact:
  - Hard to index/filter accurately; inconsistent values possible.
- Repro:
  1. Save prompt with multiple tools.
  2. Search/filter semantics depend on substring matching.
- Fix recommendation:
  - Normalize tools using enum table or relation table.
  - Add deterministic filter API using structured tool IDs.

P2-202 - Slug generation is collision-prone and non-deterministic
- Severity: `P2`
- Effort: `M`
- Evidence:
  - `src/actions/admin.ts` appends random `0-999` to sanitized title for slug.
- Impact:
  - Collision risk grows with scale.
  - URL instability impacts SEO and editorial control.
- Repro:
  1. Create many prompts with same title.
  2. Observe potential unique conflicts and random URL suffixes.
- Fix recommendation:
  - Deterministic slug pipeline with uniqueness resolver (`-2`, `-3`, ...).
  - Keep canonical slug history if titles change.

P2-203 - Analytics schema too thin for actionable product analytics
- Severity: `P2`
- Effort: `M`
- Evidence:
  - `prisma/schema.prisma` `Analytics` model only has `event`, `promptId`, `timestamp`.
- Impact:
  - Cannot analyze channel/source/session/device funnels.
- Repro:
  1. Attempt to answer "which page/source converts best for copy events".
  2. Missing data dimensions prevent analysis.
- Fix recommendation:
  - Add dimensions: `sessionId`, `referrer`, `ua`, `ipHash`, `pagePath`, `tool`.
  - Add retention/aggregation strategy.

P2-204 - Admin list pages load full datasets with no pagination
- Severity: `P2`
- Effort: `M`
- Evidence:

- Impact:
  - Response times and memory usage degrade as data grows.
- Repro:
  1. Seed large prompt/category volume.
  2. Observe heavy server render and table paint times.
- Fix recommendation:
  - Implement cursor or offset pagination.
  - Select only rendered fields; lazy-load heavy data.

P2-205 - Search architecture can overload DB and produce stale results races
- Severity: `P2`
- Effort: `M`
- Evidence:
  - `src/app/search/page.tsx` fires debounced action queries on text changes.
  - No request cancellation, pagination, or total-count API contract.
- Impact:
  - Increased DB pressure, inconsistent result ordering under latency.
- Repro:
  1. Type quickly with unstable network/latency.
  2. Observe potential stale result replacement.
- Fix recommendation:
  - Move to dedicated search API with abortable requests.
  - Add pagination, deterministic ordering, and index-backed filtering.
